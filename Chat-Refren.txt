Only content.js changed. Two fixes:

Fix 1 — The loop-breaking bug (Could not establish connection)

chrome.runtime.sendMessage() throws when the popup is closed (which it almost always is after you click Start and move away). The old notify() had no error handling, so the unhandled promise rejection was silently killing the entire async loop after country 1.

The fix is one line — .catch(() => {}) on the promise, wrapped in try-catch:

js
chrome.runtime.sendMessage({ action, ...extra }).catch(() => {});
Now a closed popup is ignored and the loop continues to country 2, 3, etc.

Fix 2 — Clipboard Document is not focused

Before writing to clipboard, we now call editor.click() before editor.focus(). The .click() counts as a real user interaction that grants document focus, which satisfies Chrome's clipboard API requirement. This should make Method A (clipboard paste) work, but Method B (execCommand) is still there as fallback since it was already working.

The real problem was that the send button is not type="submit" on the chat continuation page — it's a regular type="button" that appears dynamically. Every previous version was exclusively searching for type="submit", which only exists on the /new page for the suggestion chips.

Three fixes in this version:

1. Editor now uses data-testid="chat-input" — confirmed from the full rendered HTML you shared. This is more reliable than class-based selectors that change with deployments.

2. Send button search is now type-agnostic — searches in this order:

aria-label="Send message" (works when Claude uses it)
type="submit" icon-only in fieldset (new chat page chips area)
Last visible SVG-icon-only button inside the input container (works on chat continuation page)
Positional fallback: button closest to bottom-right of viewport
3. Enter key has correct properties — shiftKey: false, ctrlKey: false, metaKey: false explicitly set. Claude's Tiptap config intercepts plain Enter to submit, but only when shift is not held.

After send, it checks wasSent() — editor empty OR generation started — before trying the next strategy, so it won't double-send.
